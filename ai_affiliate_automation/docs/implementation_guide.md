# 実装状況と初期設定ガイド

## 1. 何をどこまで開発したか
- deterministic-first の雛形ディレクトリを作成
- 設定ファイル雛形 (`automation_config.yaml`) を追加
- 事業背景とプロンプトを分離して管理するための文書テンプレートを追加
- すぐ着手できるワークスペースREADMEを追加

## 2. 事前設定なしで動くか
最小限の雛形生成コマンドは **事前設定なしで実行可能** です。
ただし、実際の自動化運用（LLM接続、CMS連携、配信、分析）には次が必要です。

- APIキー/資格情報
- DB設定
- 媒体別ポリシーに沿った承認フロー
- 証拠キャプチャの実行環境

## 3. 初心者向けの進め方
1. `business_background.md` に事業の前提を箇条書きで記入
2. `prompt_library.md` に使うプロンプトだけを転記
3. `automation_config.yaml` で予算と禁止運用を設定
4. まず ingest/normalize から実装し、LLMは最後に接続
5. 公開は必ず QA を通し、リスク高は人手承認
